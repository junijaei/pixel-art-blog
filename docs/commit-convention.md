# 커밋 컨벤션

## 커밋 구조

```
<type>: <summary>
```

## 커밋 타입

| 타입 | 설명 |
|------|------|
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| docs | 문서 변경 |
| style | 코드 스타일/포맷 변경 (로직 변경 없음) |
| refactor | 동작 변경 없는 코드 구조 개선 |
| perf | 성능 개선 |
| test | 테스트 코드 추가 또는 수정 |
| build | 빌드 시스템 또는 의존성 변경 |
| ci | CI/CD 설정 변경 |
| chore | 유지보수 작업 |
| revert | 이전 커밋 되돌리기 |

## 커밋 메시지 규칙

- summary는 간결하고 명확하게 작성
- 명령형 어조 사용
- 불필요한 단어 지양
- 한국어로 작성

## 본문 규칙

- 불릿 포인트(`-`) 사용
- 변경 내용을 구체적으로 기술
- "무엇을 변경했는지"와 "왜 변경했는지"에 집중

## 예시

```
feat: 사용자 로그인 기능 추가

- JWT 기반 로그인 로직 구현
- 로그인 실패 시 에러 메시지 처리
- 인증 API 연동
```
